# Менеджер задач — PWA

Контрольная работа №3 по дисциплине «Фронтенд и бэкенд разработка».

Проект представляет собой прогрессивное веб-приложение для управления задачами. В приложении реализованы офлайн-режим, App Shell, Web App Manifest, WebSocket-взаимодействие, push-уведомления и напоминания.

## Цель работы

Объединить результаты практических занятий 13–18 в одном рабочем приложении:

- практика 13 — `Service Worker` и базовый офлайн-функционал;
- практика 14 — `Web App Manifest` и установка PWA;
- практика 15 — архитектура `App Shell`;
- практика 16 — `WebSocket` и push-уведомления;
- практика 17 — напоминания и действие `Отложить на 5 минут`;
- практика 18 — тестирование и оформление итогового проекта.

## Функционал

- добавление новых задач;
- удаление задач;
- отметка задач как выполненных;
- сохранение задач в `localStorage`;
- добавление задач с категорией;
- отображение счётчиков по категориям;
- создание задач с напоминанием по дате и времени;
- всплывающие уведомления в реальном времени через `Socket.IO`;
- системные push-уведомления;
- работа интерфейса и страниц в офлайн-режиме.

## Описание файлов

- `index.html` — основной App Shell приложения;
- `app.js` — клиентская логика, работа с задачами, навигацией, WebSocket и push;
- `sw.js` — `Service Worker` для кэширования, офлайн-режима и обработки уведомлений;
- `server.js` — серверная часть на `Express` и `Socket.IO`, работа с push-подписками и напоминаниями;
- `manifest.json` — манифест PWA;
- `content/home.html` — главная страница со списком задач и формами;
- `content/categories.html` — страница категорий;
- `content/about.html` — страница с описанием приложения.

## Установка и запуск

1. Открыть терминал в папке проекта.
2. Установить зависимости:

```bash
npm install
```

3. Запустить сервер:

```bash
npm start
```

4. Открыть приложение в браузере:

```text
http://localhost:3001
```

Для тестирования `Service Worker` и push локального `localhost` достаточно. При необходимости можно дополнительно использовать `HTTPS` через `mkcert`, как указано в методических материалах к практике 15.

## Проверка работоспособности

1. Запустить проект командой `npm start`.
2. Открыть `http://localhost:3001`.
3. Добавить несколько задач и убедиться, что после обновления страницы они сохраняются.
4. Проверить вкладки `Главная`, `Категории`, `О приложении`.
5. Разрешить уведомления в браузере.
6. Создать задачу с напоминанием и дождаться системного уведомления.
7. Открыть приложение в двух вкладках и проверить всплывающее уведомление через `WebSocket`.
8. Отключить сеть и убедиться, что интерфейс и закэшированные страницы продолжают открываться.

В ходе работы было разработано PWA-приложение для управления задачами, соответствующее требованиям практических занятий 13–18. Приложение поддерживает хранение данных локально, офлайн-доступ, установку как PWA, уведомления в реальном времени и push-напоминания.

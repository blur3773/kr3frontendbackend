# Менеджер задач — PWA

Progressive Web Application для управления личными задачами с офлайн-доступом, уведомлениями в реальном времени и напоминаниями.

## Технологии

- **Service Worker** — кэширование ресурсов, офлайн-режим, перехват сетевых запросов
- **Web App Manifest** — метаданные приложения, установка на домашний экран
- **App Shell** — архитектура с разделением на статический каркас и динамический контент
- **WebSocket (Socket.IO)** — двусторонняя связь в реальном времени между клиентами
- **Push API + Web Push** — системные push-уведомления даже при закрытом приложении
- **localStorage** — локальное хранение задач и их состояний
- **HTTPS (mkcert)** — безопасное соединение для работы PWA-функций
- **Express.js** — серверная часть на Node.js

## Установка и запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Генерация VAPID-ключей

```bash
npx web-push generate-vapid-keys
```

Скопируйте сгенерированные ключи и замените их в файле `server.js`:

```js
const vapidKeys = {
    publicKey: 'ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ',
    privateKey: 'ВАШ_ПРИВАТНЫЙ_КЛЮЧ'
};
```

Также замените ключ в `app.js`:

```js
const VAPID_PUBLIC_KEY = 'ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ';
```

### 3. Настройка HTTPS (для полноценного PWA)

Установите `mkcert`:

```bash
# macOS
brew install mkcert

# Linux
sudo apt install libnss3-tools
curl -JLO "https://dl.filippo.io/mkcert/latest?for=linux/amd64"
chmod +x mkcert-v*-linux-amd64
sudo mv mkcert-v*-linux-amd64 /usr/local/bin/mkcert
```

Сгенерируйте сертификат:

```bash
mkcert -install
mkcert localhost 127.0.0.1 ::1
```

Запустите сервер с HTTPS:

```bash
npx http-server --ssl --cert localhost.pem --key localhost-key.pem -p 3000
```

### 4. Запуск сервера

```bash
node server.js
```

Сервер запустится на порту `3001`.

## Функционал

### Управление задачами
- Создание новых задач через форму на главной странице
- Отметка выполнения задач (чекбоксы)
- Удаление задач
- Статистика: всего / выполнено / ожидается

### Напоминания
- Создание задач с напоминанием на определённую дату и время
- Push-уведомления при наступлении времени напоминания
- Кнопка «Отложить на 5 минут» в уведомлении
- Напоминания работают даже при закрытом приложении

### Реальное время
- WebSocket-соединение между клиентами
- При добавлении задачи в одной вкладке, другие получают уведомление
- Push-уведомления отправляются всем подписанным пользователям

### Офлайн-режим
- Полный офлайн-доступ к интерфейсу и данным
- Стратегия Cache First для статических ресурсов
- Стратегия Network First для динамических страниц
- Добавление задач работает офлайн

### Установка приложения
- Манифест с метаданными и иконками
- Установка на домашний экран через кнопку в браузере
- Запуск в отдельном окне без интерфейса браузера (standalone)

### Категории
- Страница с категориями задач (Работа, Личное, Учёба, Покупки)
- Счётчик задач по каждой категории
- Форма для добавления задачи с выбором категории

```
task-manager/
├── content/
│   ├── home.html           # Главная: формы задач и напоминаний, список задач
│   ├── about.html          # Страница «О приложении» с описанием

├── package.json            # Зависимости сервера
└── README.md               # Документация проекта
```

## Проверка работы

1. Откройте `http://localhost:3001` в браузере
2. Нажмите F12 → Application → Service Workers — убедитесь, что SW активен
3. Перейдите на вкладку Cache Storage — проверьте наличие кэшей
4. Добавьте несколько задач — они сохранятся в localStorage
5. Откройте `http://localhost:3001` во второй вкладке — добавьте задачу в первой, во второй появится уведомление
6. Включите push-уведомления кнопкой и проверьте их получение
7. Создайте напоминание — проверьте получение push-уведомления в заданное время
8. Отключите сеть — убедитесь, что приложение работает офлайн

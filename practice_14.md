# Практическое занятие 14

## Web App Manifest

Рассмотрим Web App Manifest - файл, который превращает обычное веб-приложение в прогрессивное веб-приложение (PWA), позволяя устанавливать его на устройство и настраивать внешний вид. Решение практического задания осуществляется внутри соответствующей рабочей тетради, расположенной в СДО.

### Введение

**Web App Manifest** - это JSON-файл, содержащий метаданные о вашем веб-приложении. Он позволяет браузерам и операционным системам понимать, как обрабатывать ваш сайт при добавлении на домашний экран, в каком цветовом оформлении его отображать и как запускать (в полноэкранном режиме, как отдельное приложение и т. д.).

Манифест стал ключевым компонентом прогрессивных веб-приложений (PWA), так как именно он обеспечивает интеграцию с операционной системой устройства. Без правильно настроенного манифеста ваш сайт останется обычной вкладкой в браузере, даже если все остальные PWA-функции (Service Worker, офлайн-режим) работают корректно.

Основные возможности, которые даёт манифест:
- Установка иконки на домашний экран.
- Запуск в полноэкранном режиме (без интерфейса браузера).
- Определение ориентации экрана (книжная, альбомная).
- Установка цветов темы для адресной строки и интерфейса ОС.
- Указание стартовой страницы при запуске.

### Структура манифеста

Рассмотрим основные поля файла `manifest.json`:

| Поле | Описание | Обязательность |
|------|----------|----------------|
| `name` | Полное название приложения | Рекомендуется |
| `short_name` | Короткое название (под иконкой) | Рекомендуется |
| `start_url` | URL, который открывается при запуске | Рекомендуется |
| `display` | Режим отображения (`fullscreen`, `standalone`, `minimal-ui`, `browser`) | Опционально |
| `background_color` | Цвет фона при загрузке | Опционально |
| `theme_color` | Цвет темы (адресная строка, панель задач) | Опционально |
| `description` | Описание приложения | Опционально |
| `orientation` | Ориентация экрана | Опционально |
| `icons` | Массив иконок разных размеров | Рекомендуется |
| `scope` | Область видимости PWA (какие страницы считаются частью приложения) | Опционально |

### Создание и подключение манифеста

Продолжим разработку приложения для заметок из предыдущего практического занятия. Добавим к нему манифест и необходимые мета-теги.

**Шаг 1. Поиск и подготовка иконок**

Создайте в корне проекта папку `icons` и поместите туда изображения указанных размеров. Иконки можно сгенерировать из любого изображения с помощью онлайн-инструментов, например [ConvertICO](https://convertico.com/favicon/) или [Favicon.io](https://favicon.io/). Для маскируемых иконок (с `purpose: maskable`) лучше использовать изображения с отступами, чтобы они корректно обрезались на разных платформах.

**Шаг 2. Создание файла `manifest.json`**

Поместите файл в корень проекта (обратите внимание на размерности иконок):

```json
{

    "short_name": "Заметки",
    "start_url": "/",
    "display": "standalone",
    "background_color": "#ffffff",
    "theme_color": "#4285f4",
    "description": "Простое приложение для заметок, работающее офлайн",
    "orientation": "portrait-primary",
    "icons": [
        {
            "src": "icons/favicon-16x16.png",
            "sizes": "16x16",
            "type": "image/png"
        },
        {
            "src": "icons/favicon-32x32.png",
            "sizes": "32x32",
            "type": "image/png"
        },
        {
            "src": "icons/favicon-48x48.png",
            "sizes": "48x48",
            "type": "image/png"
        },
        {
            "src": "icons/favicon-64x64.png",
            "sizes": "64x64",
            "type": "image/png"
        },
        {
            "src": "icons/favicon-128x128.png",
            "sizes": "128x128",
            "type": "image/png"
        },
        {
            "src": "icons/favicon-256x256.png",
            "sizes": "256x256",
            "type": "image/png"
        },
        {
            "src": "icons/favicon-512x512.png",
            "sizes": "512x512",
            "type": "image/png",
            "purpose": "maskable any"
        }
    ]
}
```

**Шаг 3. Подключение манифеста в HTML**

Добавьте в `<head>` файла `index.html` ссылку на манифест и дополнительные мета-теги:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
     <!-- Web App Manifest -->
    <link rel="manifest" href="/manifest.json">
    
    <!-- Для Android -->
    <meta name="mobile-web-app-capable" content="yes">
    
    <!-- Цвет темы для браузера -->
    <meta name="theme-color" content="#4285f4">
    
    <!-- Для iOS (иконка на домашний экран) -->
    <link rel="apple-touch-icon" href="/icons/icon-152x152.png">
    
    <!-- Для iOS (статус бар) -->
    <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
    <title>Заметки (13-18)</title>
    <link rel="stylesheet" href="https://unpkg.com/chota@latest">
    <link rel="icon" href="icons/favicon.ico" sizes="16x16" type="image/ico">
</head>
<body>
    <!-- ... содержимое ... -->
</body>
</html>
```

**Шаг 4. Обновление Service Worker**

Не забудьте добавить иконки в список кэшируемых ресурсов в Service Worker (`sw.js`), чтобы они были доступны офлайн:

```js
const CACHE_NAME = 'notes-cache-v2'; 
const ASSETS = [
    '/',
    '/index.html',
    '/style.css',
    '/app.js',
    '/manifest.json',
    '/icons/favicon.ico',
    '/icons/favicon-16x16.png',
    '/icons/favicon-32x32.png',
    '/icons/favicon-48x48.png',
    '/icons/favicon-64x64.png',
    '/icons/favicon-128x128.png',
    '/icons/favicon-256x256.png',
    '/icons/favicon-512x512.png'
];
```

### Тестирование и отладка

**Проверка в DevTools:**
1. Откройте приложение через локальный сервер.
2. Нажмите F12, перейдите на вкладку **Application -> Manifest**.
3. Убедитесь, что манифест загружен без ошибок, все поля отображаются, иконки доступны.
4. Проверьте, что в **Service Workers** ваш SW активирован.

### Дополнительные возможности

**Splash Screen (экран-заставка)**  
При запуске PWA на Android автоматически показывается заставка, которая формируется на основе `background_color`, `theme_color` и иконки 512x512 с `purpose: maskable`. Чтобы управлять этим, достаточно указать соответствующие поля в манифесте.

**Ориентация экрана**  
Можно зафиксировать ориентацию, например:
```json
"orientation": "portrait"
```
Допустимые значения: `portrait`, `landscape`, `natural` и др.

**Scope и Start URL**  
Эти параметры определяют, какие страницы будут считаться частью PWA. Обычно `scope` устанавливают в `/`, а `start_url` – в `/`. Можно добавить UTM-метки для аналитики: `"start_url": "/?utm_source=pwa"`.

### Практическое задание

Необходимо доработать веб‑приложение для управления списком дел (или заметок), созданное в рамках Практического занятия №13, добавив поддержку Web App Manifest. Требования:

1. Создать файл `manifest.json` с полями:
   - `name`, `short_name`, `start_url`, `display`, `background_color`, `theme_color`, `description`, `icons`.
2. Подготовить набор иконок разных размеров (минимум 3). Иконки должны быть в формате PNG.
3. Подключить манифест в HTML с помощью `<link rel="manifest">`.
4. Добавить мета-теги для улучшения интеграции с мобильными платформами.
5. Обновить Service Worker, чтобы он кэшировал иконки и файл манифеста.
6. Протестировать возможность установки приложения на компьютер. Убедиться, что после установки приложение открывается в отдельном окне.
